## Introducción (●ˇ∀ˇ●)
![imagen 1](IMAGENES/DOM%20introduccion.png) 

En el desarrollo web actual, la capacidad de crear páginas dinámicas e interactivas es ***fundamental***. Para lograrlo, los desarrolladores necesitan una forma de *acceder, modificar y actualizar el contenido, la estructura y el estilo de una página web* después de que ha sido cargada en el navegador. Es aquí donde cobra protagonismo el ***Modelo de Objeto del Documento o DOM (por sus siglas en inglés)***. En este texto, exploraremos qué es el **DOM**, cómo se relaciona con lenguajes como **JavaScript** y por qué se ha convertido en una pieza clave para la programación *web moderna*.

## Síntesis ヾ(•ω•`)o
![imagen 2 sintesis](IMAGENES/DOM%20dom%20sintesis.png) 

*Según lo visto en clase y documentado en la presentación, **el DOM significa Document Object Model*** (Modelo de Objeto de Documento). Este modelo provee una representación estructurada del documento HTML y define de qué manera se pueden modificar, tanto su estructura y estilo como su contenido. Básicamente, es una interfaz de programación que sirve para crear, cambiar o remover elementos de documentos ***HTML*** y ***XML***. También se pueden agregar eventos a esos elementos para hacer más dinámica la página. El DOM visualiza el documento de HTML como un árbol de tres nodos, siendo el ***Documento HTML*** uno de ellos. Además, ***el DOM es una API para XML***, lo que significa que es una capa de programación intermedia que representa un documento y nos permite modificarlo a través de una serie de funciones y procedimientos que sirven como medio de comunicación entre el lenguaje de programación **(por ejemplo, JavaScript)** y los contenidos del documento.

En cuanto a la **relación entre DOM y JavaScript**, se puede explicar que *JavaScript utiliza el DOM* para acceder al documento y a sus elementos. Cada elemento es parte del modelo de objeto del documento, al cual se puede acceder y manipular utilizando el ***DOM*** y un lenguaje de escritura como ***JavaScript***. El contenido de la página es almacenado en ***DOM*** y el acceso y la manipulación se hace vía ***JavaScript***, representándose aproximadamente así: ***API (web o página XML) = DOM + JS*** (lenguaje de script). Un dato importante es que el ***DOM*** fue diseñado para ser independiente de cualquier lenguaje de programación, lo que hace que la presentación estructural del documento esté disponible desde un simple y consistente ***API***. De hecho, **las directrices del DOM pueden construirse para cualquier lenguaje, como Python.**

Sobre el acceso al ***DOM***, es importante mencionar que los diferentes navegadores tienen directrices ***DOM*** distintas, pero todos los navegadores web usan el modelo de objeto de documento para hacer accesibles las páginas web al script. Cuando se crea un ***script –esté en un elemento SCRIPT*** o incluido en una página web por la instrucción de cargar un ***script–*** inmediatamente está disponible para usarlo con el **API**, accediendo así a los elementos document o window. Como ejemplo, se puede mencionar una función que crea un nuevo elemento H1, le pone texto y después lo agrega al árbol del documento. Ahora bien, respecto a los tipos de datos importantes que utiliza el **API**, se pueden mencionar algunos de los siguientes: *document* (la raíz del objeto documento), *element* (un elemento o nodo de tipo elemento), *nodeList* (una serie de elementos, como lo que devuelve el método getElementsByTagName), *attribute* (cuando un atributo es devuelto por un miembro) y *NamedNodeMap* (una serie donde los ítems son accesibles tanto por nombre o por índice).

## Reflexión (o゜▽゜)o☆
Desde mi perspectiva, lo más útil del **DOM** es que nos permite dejar de pensar en las páginas web como documentos fijos y comenzar a tratarlas como aplicaciones vivas. Sin el **DOM**, la web sería un espacio de textos e imágenes estáticas; ahora, gracias a este modelo, podemos crear interfaces que reaccionan a cada clic, que actualizan solo una parte de la pantalla o que validan formularios al instante. Me parece fascinante que el **DOM** funcione como un "puente" entre el código que escribimos y lo que ve el usuario, dándonos un control muy quirúrgico sobre cada elemento de la página. Además, el hecho de que sea independiente del lenguaje lo hace muy versátil: no estamos atados a una sola tecnología, sino que podemos aplicar la lógica del **DOM** en distintos entornos de programación. Sin duda, entender el **DOM** es esencial para cualquiera que quiera dedicarse al desarrollo web seriamente.

## Conclusión ( •̀ .̫ •́ )✧
En conclusión, el Modelo de Objeto del Documento **DOM** es mucho más que un concepto teoríco sin importancia; es la base práctica que hace posible la web dinámica que conocemos hoy. A través de su representación en árbol de nodos y su **API**, independiente del lenguaje, el **DOM** permite a desarrolladores y diseñadores acceder, modificar y extender el contenido de una página de manera eficiente. Como se ha sintetizado a lo largo de este texto, su integración con **JavaScript** es la combinación más común y poderosa para crear experiencias interactivas. Tras reflexionar sobre su utilidad, queda claro que dominar el **DOM** no solo amplía las capacidades técnicas de un programador, sino que abre la puerta a la creación de aplicaciones web más ricas, receptivas y centradas en las necesidades del usuario.
